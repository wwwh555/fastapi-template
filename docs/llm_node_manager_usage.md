# LLM Node 管理器使用说明

## 简介

LLM Node 管理器是一个 Web 界面工具，用于查看和管理后端系统中的 LLM（大语言模型）节点配置。通过该管理器，您可以：

- 查看所有 LLM 节点列表及其详细信息
- 编辑单个节点的配置
- 批量更新多个节点的配置
- 按提供商、模块、功能等条件筛选节点
- 实时同步更新数据库和 Redis 缓存

## 访问方式

### 方式一：快捷路由（推荐）
启动后端服务后，在浏览器中访问：
```
http://localhost:8000/llm-manager
```

### 方式二：静态文件直接访问
```
http://localhost:8000/i/llm_node_manager.html
```

## 功能说明

### 1. 节点列表展示

页面加载后会自动获取所有 LLM 节点并以卡片形式展示，每个卡片显示：
- 节点名称和流式标识
- 节点描述
- 提供商ID
- 模型名称
- 业务模块
- 功能名称

### 2. 搜索和筛选

**搜索框**：支持按节点名称或描述进行模糊搜索

**提供商筛选**：下拉选择指定提供商的节点

**模块筛选**：下拉选择指定业务模块的节点

### 3. 编辑单个节点

1. 点击任意节点卡片或"编辑"按钮
2. 在弹出的编辑窗口中修改节点配置
3. 可编辑字段：
   - 节点描述
   - 提供商
   - 模型名称
   - 业务模块
   - 功能名称
   - 模型参数（JSON格式）
   - 是否启用流式输出
4. 点击"保存更改"提交更新
5. 系统会同时更新数据库和 Redis 缓存

### 4. 查看节点参数

点击"查看参数"按钮可以快速查看节点的完整参数配置。

### 5. 批量更新

批量更新功能允许您一次性更新所有节点或筛选后的节点的特定字段。

**操作步骤**：

1. 点击顶部"批量更新"按钮
2. 在弹出窗口中选择要更新的字段：
   - **提供商**：统一更改所有节点的提供商
   - **模型名称**：统一更改所有节点的模型名称
   - **模型参数**：统一更改所有节点的参数配置
   - **流式输出**：统一启用或禁用流式输出
3. 勾选对应字段的复选框并填写新值
4. 点击"执行批量更新"
5. 查看更新结果统计

**注意事项**：
- 批量更新会影响所有节点（包括筛选后的节点）
- 操作不可撤销，请谨慎操作
- 系统会显示更新成功和失败的数量

### 6. 统计信息

页面顶部实时显示：
- 总节点数
- 流式节点数
- 提供商数量

## 技术细节

### API 接口

管理器通过以下后端接口进行数据交互：

- `GET /api/v1/llm/node/list` - 获取所有节点
- `GET /api/v1/llm/node/provider/list` - 获取所有提供商
- `PUT /api/v1/llm/node/update` - 更新单个节点
- `PUT /api/v1/llm/node/batch-update` - 批量更新节点

### 数据同步

每次更新操作都会：
1. 更新数据库中的节点记录
2. 同步更新 Redis 中的模型配置缓存
3. 重新加载全局 LLM 模型实例

## 常见问题

### Q: 批量更新失败怎么办？

A: 批量更新支持部分成功。系统会返回成功和失败的节点数量及详细信息。您可以：
1. 查看错误信息
2. 刷新页面重新加载数据
3. 对失败的节点单独进行编辑

### Q: 参数 JSON 格式错误？

A: 模型参数必须是有效的 JSON 格式，例如：
```json
{
  "temperature": 0.7,
  "max_tokens": 2000,
  "top_p": 0.9
}
```

系统会在保存前验证 JSON 格式，格式错误时会提示。

### Q: 节点名称可以修改吗？

A: 不可以。节点名称是节点的唯一标识，不支持修改。如需更改节点名称，需要联系后端开发人员。

### Q: 更新后多久生效？

A: 更新操作是实时的：
1. 数据库立即更新
2. Redis 缓存立即同步
3. 新的请求会立即使用新配置

## 安全建议

1. **生产环境使用**：建议在生产环境中通过身份验证保护此页面
2. **定期备份**：批量更新前建议备份当前配置
3. **权限控制**：建议限制只有管理员可以访问此页面
4. **操作日志**：建议记录所有更新操作的日志
